# novel-init：新建小说

你的任务是帮用户建立一部新小说的基础文件。用**对话方式**引导用户，一步一步问，不要一次性列出所有问题。用大白话，不用文学术语。

## 执行步骤

### Step 1：确定小说名
问用户这部小说叫什么名字（暂定名也行）。

### Step 2：了解主角
用普通人能听懂的方式问：
- 主角叫什么名字？
- 是男是女，大概多大？
- 性格是什么样的？（比如：鲁莽热血、冷静腹黑、老实被欺负但有韧劲……）
- 故事开始时，他/她的处境是什么？（普通人？废材弟子？家族落难？）
- 有没有什么特别的东西（神器、血脉、特殊体质）？没想好可以跳过。

### Step 3：了解世界
- 这个世界有没有修炼/武功？怎么分等级？
- 有没有门派、家族、势力？主角跟哪个有关系？
- 有没有什么特别的世界规则？没想好可以先写"待定"。

### Step 4：故事方向
- 大概想写一个什么故事？（一句话就行）
- 有没有特别想写的场景或者情节？
- 有没有特别想写的反派或配角？

### Step 5：文风偏好
- 喜欢什么风格？（爽文节奏快？慢热有深度？热血？阴谋诡计？）
- 有没有参考的作家或小说？
- 有没有特别不想要的东西？

### Step 6：发布信息
问用户两个问题：
- 这部小说是男频还是女频？（男频：热血/爽文/成长/战斗为主；女频：情感/人物关系/虐恋为主；不确定也可以说）
- 有没有想法让我写一段作品简介？（50-500字，用于平台上架；没想法的话说"你来写"，我根据前面的设定自动生成）

收集完后，根据整个小说设定自动生成一段作品简介草稿（即使用户提供了想法，也基于设定完善），字数控制在100-300字之间，风格要克制、有钩子，不能像广告宣传文案。

### Step 7：读者画像
根据前面收集的信息（男频/女频、题材、风格参考、目标调性），自动生成 `读者画像.md` 草稿，不需要额外问用户问题。

生成内容包括：
- 基本画像（性别/年龄层/阅读场景/阅读经历）
- 他在找什么（根据题材和风格推断）
- 他忍受不了什么（根据文风偏好推断）
- 阅读决策节点（通用模板，固定5个节点）
- 参考坐标（从用户提供的参考作品中提取）

告诉用户：读者画像是 Reader Agent 的评分依据，可以随时打开 `读者画像.md` 手动调整。

## 生成文件

收集完信息后，确认小说根目录为 `novels/[小说名]/`（相对于当前工作目录）。
创建 `chapters/` 子目录，然后生成以下文件：

### 人物本子.md
记录主角和已知人物的设定，格式清晰，以后可以追加。

### 世界规则.md
修炼体系、门派势力、世界特殊规则，未定义的写"待定"。

### 大纲.md
核心故事一句话概括 + 用户提到的情节点。

### 情节流水账.md
仅写文件说明头，内容为空（等写完第一章后自动更新）。

### 我的文风.md
用户的风格偏好、参考作品、禁忌。

### 发布信息.md
平台上架必需信息，格式如下：

```markdown
# 发布信息

## 目标读者
男频 / 女频（二选一）

## 作品简介
（100-300字，AI根据设定生成，用户可手动修改）

## 标签建议
（根据类型和风格自动生成3-5个，如：玄幻、武侠、逆袭、成长、江湖）

## 上架状态
- 平台：（待填写）
- 上架时间：（待填写）
- 更新频率：每日4章，约5000字/章
```

### 读者画像.md
根据前面收集的信息自动生成，无需额外问用户。内容包括：
- 基本画像（性别/年龄层/阅读场景）
- 他在找什么 / 忍受不了什么
- 阅读决策节点（封面→简介→第一段→第一章→前四章）
- 参考坐标（从用户提供的参考作品推断审美底线）

## 完成后

列出已创建的文件，然后说：
> 设定已建好。
> 下一步建议：
> 1. 用 `/novel-arc` 规划前10章的走向
> 2. 用 `/novel-write` 开始写第一章（前四章 Reader 评分门槛为9分，Planner 会特别关注开篇钩子）
> 3. 封面可以先去同类爆款截几张图放到 `novels/[书名]/封面参考/`，然后运行 `/novel-cover`
>
> 设定不完整没关系，写着写着补就行。
> `读者画像.md` 是 Reader Agent 的评分依据，可以随时手动调整。
