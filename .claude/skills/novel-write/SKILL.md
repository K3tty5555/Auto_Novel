# novel-write：写新章节

用户调用此 skill 时可能提供：灵感描述、已写片段、或什么都没说。
目标：输出一章约5000字，质量达标，无AI腔，前后不矛盾，读者体验过关。

---

## Phase 0：确认小说和章节号

检查 `novels/` 下的子目录：
- 只有一部 → 直接用
- 多部 → 问用户选哪部
- 查看 `chapters/` 确认下一章章节号（已有 ch003 则新章为 ch004）

**记录章节号**，后续 Reader Agent 据此判断适用门槛：
- ch001–004 → 开篇钩子阶段，Reader 门槛 9 分
- ch005–010 → 早期留存阶段，Reader 门槛 8.5 分
- ch011+ → 常规阶段，Reader 门槛 8 分

---

## Phase 1：ContextLoader

启动 **general-purpose agent** 读取以下文件，返回结构化摘要：

```
novels/[小说名]/人物本子.md
novels/[小说名]/世界规则.md
novels/[小说名]/情节流水账.md（最后10条）
novels/[小说名]/我的文风.md
novels/[小说名]/大纲.md
novels/[小说名]/读者画像.md
novels/[小说名]/chapters/ 中最新章节的最后500字
```

返回格式：
- 【人物速查】主要人物名+核心特征
- 【世界规则速查】最关键的几条规则
- 【上章结尾】最后一幕状态
- 【情节禁区】绝对不能出现的矛盾点
- 【当前大纲节点】现在处于大纲哪个位置
- 【文风要求】核心偏好关键词
- 【读者画像摘要】目标读者是谁、最在意什么

---

## Phase 2：Planner Agent

启动 **general-purpose agent**，输入 Phase 1 摘要 + 用户灵感/片段（如有）。

任务：生成本章分场大纲：
- 5~8个场景，每个场景1-2句话
- 明确本章爽点/情绪高潮在哪个场景
- 结尾留下钩子（让读者想看下一章）
- 检查是否与情节禁区冲突
- **如果是 ch001–010，额外说明**：本章的"读者留下理由"是什么

输出格式：
```
本章标题（建议）：xxx
本章核心：[一句话]
爽点/高潮：[第几场景]
读者留下理由：[ch001-010 必填，ch011+ 可选]

场景一：[地点/人物/发生了什么]
场景二：...
结尾钩子：[下章悬念]
```

---

## Phase 3：Writer Agent

启动 **general-purpose agent**，输入 Phase 1 摘要 + Phase 2 大纲 + 用户片段（如有）。

任务：按大纲写完整章节：
- 总字数约5000字（4500~5500均可）
- 如用户提供了片段，从片段末尾接续，不要重写片段
- 对话符合各人物性格
- 战斗/修炼场景要有画面感
- 禁止滥用：连续"只见"超3次、"不禁"、"顿时"密集出现、"心中暗道"泛滥
- 结尾要有停顿感

---

## Phase 4：Critic Agent

启动 **general-purpose agent** 评分（0-10分制）：

| 维度 | 评分 | 具体问题 |
|------|------|---------|
| 连贯性 | x/10 | 与已有设定有无矛盾 |
| 节奏感 | x/10 | 动静比例是否合理 |
| 人物声音 | x/10 | 对话/行为符合性格吗 |
| 类型感 | x/10 | 玄幻/武侠爽感是否到位 |
| 自然度 | x/10 | AI腔是否严重 |

- 平均分 ≥ 8：进入 Phase 5
- 平均分 < 8：列出具体问题段落（引用开头几个字）+ 修改建议，进入 Phase 4.5

---

## Phase 4.5：Critic-Rewriter（最多2轮）

启动 **general-purpose agent**，输入草稿全文 + Critic 问题列表。

任务：**只修改被标记段落**，其余原样保留。

修改后重新 Phase 4：
- 轮次2不论分数如何，直接进入 Phase 5（记录遗留问题）

---

## Phase 5：Reader 评审（多Agent红蓝对抗）

这是独立于 Critic 的第二道关卡。Critic 评创作质量，Reader 评读者体验。

调用 **novel-reader skill（内嵌模式）**，传入章节文本和小说名。

novel-reader 内部并行启动：
- 首段守门人（只看第一段，预测+判决）
- 节奏扫描仪（标出拖沓点）
- 挑刺专员（找最致命的一个问题）
- 竞品对照员（有竞品参考时）
- 专项核查员（ch001–010：跑开篇/留存清单）

裁判Agent综合输出三档判决：
- **发出去 ✓** → 进入 Phase 6
- **改一处 ⚠️** → 进入 Phase 5.5
- **重写 ✗** → 进入 Phase 5.5（按重写级别处理）

---

## Phase 5.5：Reader-Rewriter（最多1轮）

启动 **general-purpose agent**，输入章节草稿全文 + 裁判报告中的具体修改点。

**判决为"改一处"时**：
- 只修改裁判指定的那一处，其余原样保留
- 修改重点：首段守门人的划走原因 > 挑刺专员的致命问题 > 节奏扫描仪的拖沓点

**判决为"重写"时**：
- 识别根本问题（结构性缺陷还是方向错误）
- 针对根本问题重写，不是全文重来——定位到具体场景或段落级别

修改完成后**直接进入 Phase 6**，不再重新评审（避免无限循环）。

将 Reader 判决和遗留问题记录在汇报中。

---

## Phase 6：保存 + 更新记录

保存章节文件：
```
novels/[小说名]/chapters/ch{NNN}_{标题}.md
```

文件头：
```markdown
# 第X章 [章节标题]

> 字数：约XXXX字 | 写作日期：YYYY-MM-DD

[正文]
```

在 `情节流水账.md` 末尾追加：
```
[chXXX] [标题]
- 主要事件：
- 新出场人物：（如无写"无"）
- 情节转折：（如无写"无"）
- 结尾状态/钩子：
```

---

## Phase 7：汇报

```
章节已保存：novels/[小说名]/chapters/ch[NNN]_[标题].md
字数：约[XXXX]字

Critic 评分（创作质量）：
  连贯性 X / 节奏感 X / 人物声音 X / 类型感 X / 自然度 X
  平均：X.X 分

Reader 评审（读者体验）：
  判决：[发出去 ✓ / 改一处已修改 ✓ / 重写已修改 ✓]
  致命问题：[Agent C 结论，或"无"]
  [若 ch001-010] 专项：[X]/[7或5] 项通过

遗留问题（如有）：
  · [问题描述]

[提示是否继续写下一章]
```

---

## 特殊情况

**人物本子未定义的人物出现**：写完后提醒用户补充设定。

**用户灵感与当前情节矛盾**：先告知矛盾，询问是否坚持或调整方向。

**大纲当前阶段已写完**：提示用 `/novel-arc` 规划下一段，再继续写。

**读者画像.md 不存在**：Reader Agent 跳过，在汇报中提示创建读者画像以启用 Reader 环节。
