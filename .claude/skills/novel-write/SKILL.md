# novel-write：写新章节

用户调用此 skill 时可能提供：灵感描述、已写片段、或什么都没说。
目标：输出一章约5000字，质量达标，无AI腔，前后不矛盾。

---

## Phase 0：确认小说和章节号

检查 `novels/` 下的子目录：
- 只有一部 → 直接用
- 多部 → 问用户选哪部
- 查看 `chapters/` 确认下一章章节号（已有 ch003 则新章为 ch004）

---

## Phase 1：ContextLoader

启动 **general-purpose agent** 读取以下文件，返回结构化摘要：

```
novels/[小说名]/人物本子.md
novels/[小说名]/世界规则.md
novels/[小说名]/情节流水账.md（最后10条）
novels/[小说名]/我的文风.md
novels/[小说名]/大纲.md
novels/[小说名]/chapters/ 中最新章节的最后500字
```

返回格式：
- 【人物速查】主要人物名+核心特征
- 【世界规则速查】最关键的几条规则
- 【上章结尾】最后一幕状态
- 【情节禁区】绝对不能出现的矛盾点
- 【当前大纲节点】现在处于大纲哪个位置
- 【文风要求】核心偏好关键词

---

## Phase 2：Planner Agent

启动 **general-purpose agent**，输入 Phase 1 摘要 + 用户灵感/片段（如有）。

任务：生成本章分场大纲：
- 5~8个场景，每个场景1-2句话
- 明确本章爽点/情绪高潮在哪个场景
- 结尾留下钩子（让读者想看下一章）
- 检查是否与情节禁区冲突

输出格式：
```
本章标题（建议）：xxx
本章核心：[一句话]
爽点/高潮：[第几场景]

场景一：[地点/人物/发生了什么]
场景二：...
结尾钩子：[下章悬念]
```

---

## Phase 3：Writer Agent

启动 **general-purpose agent**，输入 Phase 1 摘要 + Phase 2 大纲 + 用户片段（如有）。

任务：按大纲写完整章节：
- 总字数约5000字（4500~5500均可）
- 如用户提供了片段，从片段末尾接续，不要重写片段
- 对话符合各人物性格
- 战斗/修炼场景要有画面感
- 禁止滥用：连续"只见"超3次、"不禁"、"顿时"密集出现、"心中暗道"泛滥
- 结尾要有停顿感

---

## Phase 4：Critic Agent

启动 **general-purpose agent** 评分（0-10分制）：

| 维度 | 评分 | 具体问题 |
|------|------|---------|
| 连贯性 | x/10 | 与已有设定有无矛盾 |
| 节奏感 | x/10 | 动静比例是否合理 |
| 人物声音 | x/10 | 对话/行为符合性格吗 |
| 类型感 | x/10 | 玄幻/武侠爽感是否到位 |
| 自然度 | x/10 | AI腔是否严重 |

- 平均分 ≥ 8：进入 Phase 6
- 平均分 < 8：列出具体问题段落（引用开头几个字）+ 修改建议，进入 Phase 5

---

## Phase 5：Rewriter Agent（最多2轮）

启动 **general-purpose agent**，输入草稿全文 + Critic 问题列表。

任务：**只修改被标记段落**，其余原样保留。

修改后重新 Phase 4：
- 轮次2不论分数如何，直接进入 Phase 6（记录遗留问题）

---

## Phase 6：去AI味

对最终版本调用 **Humanizer-zh skill**。

重点：
- 开头不能有"在那……的世界里"套路
- 去掉过度整齐的排比和三段式
- 句子长短要参差不齐

---

## Phase 7：保存 + 更新记录

保存章节文件：
```
novels/[小说名]/chapters/ch{NNN}_{标题}.md
```

文件头：
```markdown
# 第X章 [章节标题]

> 字数：约XXXX字 | 写作日期：YYYY-MM-DD

[正文]
```

在 `情节流水账.md` 末尾追加：
```
[chXXX] [标题]
- 主要事件：
- 新出场人物：（如无写"无"）
- 情节转折：（如无写"无"）
- 结尾状态/钩子：
```

---

## Phase 8：汇报

告诉用户：
- 章节保存路径
- 本章字数
- Critic 最终各维度评分
- 遗留问题（如有）
- 提示是否继续写下一章

---

## 特殊情况

**人物本子未定义的人物出现**：写完后提醒用户补充设定。

**用户灵感与当前情节矛盾**：先告知矛盾，询问是否坚持或调整方向。

**大纲当前阶段已写完**：提示用 `/novel-arc` 规划下一段，再继续写。
