# novel-reader：红蓝对抗读者评审

用法示例：
```
/novel-reader 别给我队友 ch001          ← 评一章
/novel-reader 别给我队友 ch001-004      ← 评前四章（自动触发连读专项）
/novel-reader 别给我队友 --cover        ← 评封面
```

> **架构说明**：4个专项Agent并行运行，最后由裁判Agent综合。输出三档判决而非数字分——
> 因为"8.5 vs 9.0"的精度是假的，"发/改/重写"是真实决策。

---

## Phase 0：加载上下文

**0.1 读取读者画像**

读取 `novels/[小说名]/读者画像.md`，提取：
- 读者核心诉求
- 忍受不了什么
- 参考坐标（审美底线）

如文件不存在，跳过竞品对照员，其余Agent继续运行，汇报末尾提示补充画像。

**0.2 检查竞品参考文本**

```bash
ls novels/[小说名]/竞品参考/ 2>/dev/null
```

- 有文件 → 竞品对照员正常运行
- 无文件 → 竞品对照员跳过，汇报末尾提示添加竞品

**竞品文本提取规则**：
- 如果文件是完整小说（含多章），截取第一个"第X章"标题到第二个"第X章"标题之间的内容作为参照
- 如果文件本身就只有一章，直接使用全文
- 参照文本长度控制在3000字以内，超过则截取前3000字

**0.3 收集待评章节**

按章节号升序收集目标 `.md` 文件，记录章节数量（判断是否触发连读专项）。

**0.4 检查评审扩展**

检查 `novels/[小说名]/评审扩展.md` 是否存在：
- 存在 → 读取文件，提取扩展Agent指令（文件内每个 `## Agent X：` 区块即一个扩展Agent），Phase 1 中与 A-E **同步并行**启动
- 不存在 → 跳过，Phase 1 正常运行

---

## Phase 1：并行启动专项Agent

> **必须并行**——所有Agent（含扩展）同时启动，不等待彼此结果。
> 基础Agent A-E 始终运行；扩展Agent（如 Agent F）仅在 0.4 检测到扩展文件时启动。

---

### Agent A：首段守门人

**只给它看第一段**（约前150字，截到第一个段落结束）。

Agent 指令：
```
你在刷网文推荐流，看到了下面这段文字。
你不知道后面有什么，你有随时划走的权利。

[第一段文本]

先做一件事：基于这段话，你预测这个故事接下来会发生什么？主角是什么性格？
然后决定：你会继续读这本书，还是划走？

输出格式：
预测：[你对故事走向和主角性格的预判，2-3句]
决定：继续 / 划走
理由：[一句话，越具体越好，不接受"感觉不错"或"感觉平平"这种回答]
```

拿到决定后，给它看完整章节，再问：
```
你刚才预测的内容，哪里应验了？哪里没有？
没应验的地方，是让你有惊喜，还是让你困惑？
```

**输出记录**：首段判决（继续/划走）+ 预测偏差点（惊喜/困惑）

---

### Agent B：节奏扫描仪

读取完整章节。

Agent 指令：
```
你是一个读网文读了三年的人，书架收了200本。
读下面这章，不评价好坏。
你只做一件事：每当你产生"想跳过这段"的念头，就记下来。

输出格式（每个拖沓点一条）：
> [引用让你想跳过的那段的第一句话]
原因：[一句话，为什么想跳]

如果全章没有任何想跳过的地方，输出"无"。
```

**输出记录**：拖沓点列表（引用+原因），或"无"

---

### Agent C：挑刺专员（红队）

读取完整章节 + 读者画像。

Agent 指令：
```
你是一个编辑，任务是找这章最致命的一个问题。

不是小问题，不是"可以更好"——是"如果不改，这本书大概率在第一章就流失读者"的问题。
如果真的没有致命问题，直接说"没有"。

只说一个问题。不说其他的。

输出格式：
致命问题：[具体描述，要精确到哪个段落或哪种写法]
为什么致命：[一句话]
```

**输出记录**：致命问题（或"没有"）

---

### Agent D：竞品对照员（有竞品参考时运行）

读取竞品第一章 + 本章。

Agent 指令：
```
先读这篇文章（竞品ch001）：
[竞品文本]

读完，记下你的第一反应。

再读我们的ch001：
[本章文本]

现在回答：
1. 如果这两章同时出现在推荐页，你先点哪个？为什么？
2. 我们这章比竞品强在哪一点？
3. 我们这章比竞品弱在哪一点？

不打分，只做比较。
```

**输出记录**：相对强弱各一点

---

### Agent E：专项核查员（ch001–010 时运行）

读取完整章节，根据章节号执行对应清单。

**ch001–004 开篇钩子清单（7项，必须每项给 Y/N + 一句说明）**：

```
□ 第一段是否以具体场景/动作/对话开始（而非背景介绍）？
□ 主角在第一章是否有一个让读者记住他的具体瞬间？
□ 第一章结尾是否给了翻下一章的理由？
□ 前四章里有没有"哦这本有意思"的时刻？
□ 世界观是通过行动渗透的，还是感觉在读设定说明书？
□ 读完这章，清楚主角想要什么吗？
□ 到这章末尾，会追更吗？
```

**ch005–010 早期留存清单（5项）**：

```
□ 每章是否都有至少一个有效钩子（不是全在铺垫）？
□ 主角成长是否有迹可循（代价感、努力感）？
□ 世界观扩展是否自然（没有突然塞入大量设定）？
□ 主线是否在推进（读完感觉有进展）？
□ 读完这段，还想继续追吗？
```

**输出记录**：清单通过项数 + 未通过项的具体说明

## Phase 2：裁判 Agent 综合

**等所有并行Agent完成后**，启动裁判Agent，输入：
- Agent A 的首段判决 + 预测偏差
- Agent B 的拖沓点列表
- Agent C 的致命问题
- Agent D 的竞品比较（如有）
- Agent E 的专项清单（如适用）
- 扩展Agent（如 Agent F）的报告（如有）

裁判 Agent 指令：
```
你是主编，收到了以下几份评审报告：

[各Agent报告]

现在给出最终判决，三选一：
- 发出去 ✓：现在可以发，问题不影响读者留存
- 改一处 ⚠️：有一个明确问题需要先改，改完可发
- 重写 ✗：结构性问题，改局部解决不了

输出格式：
判决：[发出去 / 改一处 / 重写]
主要依据：[最影响判决的一条报告，引用具体内容]
如果"改一处"：具体改什么（精确到段落）
如果"重写"：根本问题是什么
```

---

## Phase 3：连读专项（章节数 ≥ 3 时追加）

等 Phase 2 完成后，单独启动连读 Agent。

Agent 指令：
```
读完以下 [N] 章，从连读视角评估。

[所有章节文本]

回答以下问题（每项必须给出具体章节号或引用，不接受泛泛而谈）：

1. 节奏多样性：这几章的动/静分布是否有变化？哪章是明显的反差点或疲软点？

2. 钩子类型：每章结尾钩子是否不重复？
   逐章速记：ch[NNN]=[类型], ...
   （类型：危机未解/信息差/情绪共鸣/反转/物件伏笔/人物悬念）

3. 情绪曲线：连读完有没有情绪起伏？从第几章开始感觉"温度"持续太平？

4. 主线感知：读完这批章节，主线是否在推进？停滞从第几章开始？

5. 疲劳点：连读下来，从第几章开始感觉"这本可能不太行"？没有则说"没有"。

6. 记忆点：读完这几章，脑子里能浮现几个具体的场景或台词？举出1-2个，或说"几乎没有"。
```

## 内嵌模式（被 novel-write / novel-cover 调用时）

返回给调用方的结构化结果：
```
verdict: 通过 / 需修改 / 需重写
if 需修改:
  fix: [具体修改点，一条]
if 需重写:
  reason: [根本问题]
fatal_issue: [Agent C 的致命问题，或 null]
```

`novel-write` 据此判断是否触发 Reader-Rewriter（verdict = 需修改时）。

## 封面评审（--cover 参数）

封面不走多 Agent，保持单 Agent 评审：

读取 `novels/[小说名]/封面.png`，启动 Agent，读者视角评价：

```
【3秒扫描】
第一反应：会点 / 可能会 / 不会
原因：

【逐项】
□ 书名清晰：Y/N
□ 主视觉有记忆点：Y/N（说明是什么）
□ 风格符合类型期待：Y/N
□ 视觉冲击：Y/N

【判决】
发出去 ✓ / 改一处 ⚠️（改什么）/ 重做 ✗（根本问题）
```

---

## 注意事项

- 所有专项 Agent **必须并行启动**，不要串行等待（含扩展Agent）
- Agent B 节奏扫描仪的每条拖沓点**必须引用原文**，不接受"整体节奏偏慢"
- Agent C 挑刺专员**只说一个问题**，强迫它找最致命的而非列清单
- 裁判 Agent 的判决**必须明确三选一**，不接受"介于改一处和重写之间"
- 连读专项钩子类型速记**必须逐章列出**
- 没有竞品参考时跳过 Agent D，但在汇报末尾提示添加
- 扩展Agent（如 Agent F）的致命级问题**必须传入裁判Agent**，裁判可据此升级判决
