# 小说生成器 — AI工作指令

## 项目定位
这是一个辅助网文写作的系统。用户每天产出4章×5000字，发布在网络平台。

## 你的工作方式

### 读取上下文的优先级
每次写作前，必须按顺序读取：
1. `novels/[小说名]/人物本子.md` — 人物设定，绝对不能违背
2. `novels/[小说名]/世界规则.md` — 世界观和规则，不能违背
3. `novels/[小说名]/情节流水账.md` — 已发生的事，不能矛盾
4. `novels/[小说名]/我的文风.md` — 写作风格要求
5. `novels/[小说名]/chapters/` 最新一章结尾 — 保证衔接自然

### 写作完成后，必须做
- 在 `情节流水账.md` 末尾追加本章摘要（3-5行，记关键事件、新出场人物、情节转折）
- 章节文件保存到 `chapters/` 目录，命名格式：`ch{三位数字}_{章节标题}.md`

### 质量标准（Critic评分维度）
- **连贯性**：与人物本子、世界规则、已有情节无矛盾
- **节奏感**：动静结合，不能全是打斗或全是废话
- **人物声音**：每个人说话方式要符合其性格
- **类型感**：玄幻/武侠该有的爽点、氛围要到位
- **自然度**：读起来像人写的，不能有明显AI腔

平均分 < 8 分时，必须针对性修改，最多迭代2轮。

### 禁止事项
- 禁止在人物本子未定义的情况下，给人物加新的重要能力或背景
- 禁止改变世界规则.md中已确定的设定
- 禁止跳过情节流水账.md直接写（会导致前后矛盾）
- 禁止输出明显的AI套话（"只见"连续出现3次以上、"不禁"、"顿时"滥用等）

## 文件路径约定
```
novels/
  [小说名]/
    人物本子.md       ← 所有重要人物的设定
    世界规则.md       ← 修炼体系、门派、世界规则
    大纲.md           ← 全书主线（可以很粗）
    情节流水账.md     ← AI自动维护的已发情节记录
    我的文风.md       ← 用户风格偏好
    发布信息.md       ← 平台上架必需：目标读者、作品简介、标签
    chapters/         ← 已输出的章节
```

## 发布信息 SOP
每部小说必须包含 `发布信息.md`，内容：
- **目标读者**：男频 or 女频
- **作品简介**：100-300字，克制有钩子，不能像宣传文案
- **标签建议**：3-5个类型标签
- **更新频率**：每日4章，约5000字/章

此文件由 `/novel-init` 自动生成草稿，用户可随时手动修改。

## Skill 使用说明
- `/novel-init` — 新建一部小说，引导填写基础设定
- `/novel-write` — 写新章节（支持传入灵感或片段）
- `/novel-arc` — 规划下一段故事弧（5-10章大纲）
- `/novel-recap` — 生成当前进度摘要
